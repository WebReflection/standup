You might think I am glitching in here but every bug I've recently filed against *FireFox* leads there ... there is an issue with whoever edited the original issue topic, because that was supposed to be "*MessagePort.postMessage for received MessagePorts will fail to send messages if blocking APIs (Sync XHR, Atomics) are used prior to the Entangling state machine stabilizing; workaround is to wait for receipt of a message*" ... and I've just realized I need to follow up ...
